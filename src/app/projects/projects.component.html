Pay less eat more (PLAM)


This project is about paying less for food. Say that you want to eat spaghetti bolognese, then you can search for it in my web app to show you where the cheapest option is, based on the stores you have chosen.
For example, pasta costs 10 kr in Netto and 15 kr in Fakta. Then you can see that you can save 5 kr by buying it in Netto.
Meat costs 50 kr in Fakta, but 60 kr in Netto, so the app would choose the cheapest option. The only downside is that you have to go to each store to buy separate items.

This project has been in the works for some time now, and I keep hitting a roadblock called ghosting. I wrote to Tjek (e-offer app), COOP, and Salling Group, but no one has responded to me, so I have to find a new way to get the data I need.
Since Tjek does offer an API, I have tried to use their own API without documentation, but I have not been able to get it to work fully.
The closest I got is 25 items from each store.

Salling Group does have an API with documentation, but it only works for "BilkaToGo" and not for offers. This is a start, and I will try to get it to work.
Salling Group also has food waste initiatives that I would like to implement, though there has to be a disclaimer of some sort, since someone could already have taken the item.
