Truth or bullshit!

Truth or Bullshit is a Danish card game about common and uncommon knowledge, where players guess whether a statement is true or just bullshit.
I created an Android app for the game and wrote a Python script to read all the cards and generate a JSON file for the app.
The app is not available on the Play Store because I don't have the rights to the game, but the code is publicly available for anyone to use.
Due to copyright restrictions, I cannot share the JSON file.
I started this project because I wanted to bring the game to a New Year's Eve party. However, since other family members had plans to take the physical game with them, I decided to turn it into an app.
At the same time, I had always wanted to try coding in Python, so I saw this as a great small project to get started.
There are probably better or more optimized ways to do this, but this was a learning project for me. It took me about 14 hours in total - from taking pictures of the cards to completing the app.
The Kotlin part of the project is available here.
The Python part of the project is available here.
Packages used:

  • Pytesseract: for reading images
  • CV2: for editing images to improve readability
  • Json: for generating the JSON file
  • Numpy: for sharpening the images

Pay less eat more (PLAM (pending name))


This project is about paying less for food. Say that you want to eat spaghetti bolognese, then you can search for it in my web app to show you where the cheapest option is, based on the stores you have chosen.
For example, pasta costs 10 kr in Netto and 15 kr in Fakta. Then you can see that you can save 5 kr by buying it in Netto.
Meat costs 50 kr in Fakta, but 60 kr in Netto, so the app would choose the cheapest option. The only downside is that you have to go to each store to buy separate items.

This project has been in the works for some time now, and I keep hitting a roadblock called ghosting. I wrote to Tjek (e-offer app), COOP, and Salling Group, but no one has responded to me, so I have to find a new way to get the data I need.
Since Tjek does offer an API, I have tried to use their own API without documentation, but I have not been able to get it to work fully.
The closest I got is 25 items from each store.

Salling Group does have an API with documentation, but it only works for "BilkaToGo" and not for offers. This is a start, and I will try to get it to work.
Salling Group also has food waste initiatives that I would like to implement, though there has to be a disclaimer of some sort, since someone could already have taken the item.
